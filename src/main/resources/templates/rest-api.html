RadCheck has its own REST API!

You can use RadCheck to search locations with an address, latitude/longitude coordinates, or find and refresh results within the database as a JSON reply. The following is the basic anatomy of a API query:

https://www.radcheck.app/api/query


However, the API needs parameters to know what to search for! If you know the ID of the location you want to find, use the query parameter id:

https://www.radcheck.app/api/query?id=1


A database location's results can be refreshed within the API by using the boolean query parameter refresh, and it accepts values of "1", "on", "true" and "yes" to initiate a refresh:

https://www.radcheck.app/api/query?id=2&refresh=1


Searches can be made with addresses using the parameter address. No special encoding is required, the API will handle the request.

https://www.radcheck.app/api/query?address=Big Ben, London, UK


Searches can also be made with coordinates using lat for latitude and lon for longitude:

https://www.radcheck.app/api/query?lat=38.624796&lon=-90.184779


One very big difference between the visual interface of RadCheck and its API is that search distance in meters can be defined as a query parameter. Just add distance to a search using an address or coordinates to change it from the default of 1000 meter radius to whatever you want!

https://www.radcheck.app/api/query?distance=30&address=Times Square, Manhattan, New York, NY


If the API is given parameters that are incompatible or missing, it will return a HTTP 400 status along with a reason why it happened. Now you can use RadCheck with your own website or application!
